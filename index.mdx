This project is a hands-on Java exercise that models a music academy using core object-oriented programming concepts: **inheritance**, **abstract classes**, **interfaces**, and **enums**. It covers managing instruments, courses, students, and teachers — all with robust input validation and enrollment logic.

## What this project covers

<Steps>
  <Step title="Academy management">
    The `Academia` class manages a catalog of instruments (pianos and guitars), a list of courses, and a registry of teachers — all keyed by unique codes or DNI identifiers.
  </Step>
  <Step title="Instrument catalog">
    `Piano` and `Guitarra` both extend the abstract `Instrument` class. Each instrument has a rental price, a unique code, and type-specific attributes (key count for piano, string type for guitar).
  </Step>
  <Step title="Course enrollment">
    `Curs` enforces level-based enrollment rules using the `Nivell` enum (`INICIAL`, `INTERMEDI`, `AVANÇAT`). Advanced courses require students to be 16 or older.
  </Step>
  <Step title="Notifications">
    When a student successfully enrolls, `ServeiNotificacions.enviarConfirmacio()` automatically prints a confirmation with the student's name and course name.
  </Step>
</Steps>

## Key design patterns

<CardGroup cols={2}>
  <Card title="Inheritance & abstraction" icon="sitemap" href="/domain/instruments">
    `Piano` and `Guitarra` extend the abstract `Instrument` class, which enforces a `descriure()` method on all instruments.
  </Card>
  <Card title="Interface contract" icon="file-contract" href="/reference/avaluable">
    `Curs` implements the `Avaluable` interface, committing to a `calcularNotaFinal()` method for grade calculation.
  </Card>
  <Card title="Enum levels" icon="list-ol" href="/reference/nivell">
    The `Nivell` enum defines three course levels and drives enrollment age-restriction logic.
  </Card>
  <Card title="Validation rules" icon="shield-check" href="/guides/validation-rules">
    Every factory method validates its inputs — null checks, empty string checks, duplicate code checks, and positive-value checks.
  </Card>
