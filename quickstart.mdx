## Prerequisites

- **Java JDK 8 or later** installed and available on your `PATH`. Verify with:

```bash
java -version
javac -version
```

Both commands should print a version number. If either is missing, install the JDK for your platform from [adoptium.net](https://adoptium.net) or your system package manager.

## Get the source

Clone the repository or download the zip from GitHub:

```bash
git clone https://github.com/Zafiro2004/EjercicioImplementacionRepaso.git
cd EjercicioImplementacionRepaso
```

All source files live under `src/`:

```
src/
├── Academia.java
├── Alumne.java
├── Avaluable.java
├── Curs.java
├── Guitarra.java
├── Instrument.java
├── Main.java
├── Nivell.java
├── Piano.java
├── Profesor.java
└── ServeiNotificacions.java
```

## Compile and run

<Steps>
  <Step title="Create an output directory">
    Create a directory to hold the compiled `.class` files:

    ```bash
    mkdir -p out
    ```
  </Step>

  <Step title="Compile all source files">
    Compile every `.java` file in `src/` and place the output in `out/`:

    ```bash
    javac -d out src/*.java
    ```

    If compilation succeeds you will see no output and `out/` will contain `.class` files for every class. Any compilation errors will be printed to stderr with the file name and line number.
  </Step>

  <Step title="Run the test runner">
    Execute `Main` with the compiled classes on the classpath:

    ```bash
    java -cp out Main
    ```
  </Step>
</Steps>

## Source bugs — fix before compiling

<Warning>
The source code as committed contains **compile-time errors** that must be fixed before `javac` will succeed:

1. **`Curs.calcularNotaFinal` return type mismatch** — The method is declared `public boolean calcularNotaFinal()` but the `Avaluable` interface requires `double`. The return statement also has a syntax error (`return 0.0}` — missing semicolon). Fix:

   ```java
   @Override
   public double calcularNotaFinal() {
       return 0.0;
   }
   ```

2. **`Curs` constructor arity mismatch** — The only `Curs` constructor requires four arguments (`codi`, `nom`, `nivell`, `instrument`), but `Main.java` constructs `Curs` objects with three arguments (no `Instrument`). Either add a three-argument constructor to `Curs`, or update all call sites to pass an `Instrument`.
</Warning>

Once both errors are fixed, compilation and execution proceed normally.

## What each test group covers

| Test method | Classes exercised | What is checked |
|---|---|---|
| `provarAcademiaAfegirPiano` | `Academia`, `Piano` | All null/empty/zero/negative guard conditions on `afegirPiano`, plus duplicate `codi` rejection |
| `provarAcademiaAfegirGuitarra` | `Academia`, `Guitarra` | Same guards for `afegirGuitarra`, plus null/empty `tipusCordes` |
| `provarAcademiaAfegirCurs` | `Academia`, `Curs`, `Nivell` | Null/empty guards, duplicate `codi`, and null `Nivell` rejection |
| `provarAcademiaAfegirProfesor` | `Academia`, `Profesor` | Null/empty guards on `nom`, `dni`, `especialitat`, plus duplicate DNI rejection |
| `provarAcademiaGetInstrument` | `Academia`, `Instrument` | Index-based retrieval returns the correct object; negative and out-of-range indices return `null` |
| `provarAcademiaGetProfesorIEliminar` | `Academia`, `Profesor` | DNI-based lookup, successful deletion, idempotent deletion attempt |
| `provarAlumne` | `Alumne` | `getNom()`, `getDni()`, and `getEdat()` return the values passed to the constructor |
| `provarProfesor` | `Profesor` | `getDni()` returns the value passed to the constructor |
| `provarCursMatricular` | `Curs`, `Alumne`, `Nivell` | Valid enrollment, duplicate DNI guard, null guard, age restriction for `AVANÇAT`, null `Nivell` behavior |
| `provarCursCalcularNotaFinal` | `Curs`, `Avaluable` | Current return value of the `calcularNotaFinal` implementation |

<Note>
  **Two known issues in the current source:**

  1. **`calcularNotaFinal` is not implemented.** `Curs.calcularNotaFinal()` currently returns `0.0` (the body contains a syntax error: `return 0.0}` with a misplaced brace). The test in `provarCursCalcularNotaFinal` is written to assert `== 0.0` precisely because that is the current behaviour — it is not asserting a correct grade calculation.

  2. **`validarAlumne` has an inverted condition for `AVANÇAT`.** In `Curs.java`, the guard is written as:

     ```java
     return this.nivell != Nivell.AVANÇAT && alumne.getEdat() >= 16;
     ```

     This means a student aged 16 or over is **rejected** when the level is `AVANÇAT` (because `nivell != AVANÇAT` is `false`) and **accepted** in every other level regardless of age. The intended logic is the opposite: only allow enrollment in `AVANÇAT` courses if the student is at least 16. The tests `provarCursMatricular` will show `[FAIL]` for the two AVANÇAT age checks until this condition is corrected.
</Note>
